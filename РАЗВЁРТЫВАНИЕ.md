# DubaiCost — запуск на своём компьютере (localhost)

Инструкция для заказчика. Windows + PowerShell. Есть два пути:

- **Путь А (рекомендуется)** — восстановить готовую БД из дампа и запустить.
  Быстро (минуты), никакого скачивания данных, точное рабочее состояние.
- **Путь Б** — собрать данные с нуля (нужен интернет и время, ~десятки минут).

---

## Что вам передали

1. **Репозиторий** проекта (эта папка).
2. Файл **`delivery/dubaicost.dump`** (~30 МБ) — готовая база (здания, цены,
   аренда, cooling, POI, диаспоры). Передаётся **отдельно** (в git его нет).
   Положите его в папку `delivery/` внутри проекта.

## Что установить один раз

| Компонент | Версия | Ссылка |
|---|---|---|
| PostgreSQL + PostGIS | 16 | postgresql.org/download/windows (в Stack Builder отметить PostGIS) |
| Python | 3.12 | python.org |
| Node.js | 18+ | nodejs.org |

## Путь А — восстановить готовую БД и запустить

### 1. Восстановить базу из дампа

Из корня проекта в PowerShell:

```powershell
cd C:\путь\к\3D_dubaicost
.\scripts\restore_db.ps1 -DbPassword "ПридумайтеПароль"
```

Скрипт спросит пароль суперпользователя `postgres` (задан при установке
PostgreSQL), создаст роль и базу, включит PostGIS и зальёт дамп. В конце
покажет число зданий и метрик — если там ~45890 зданий, всё хорошо.
(Одно предупреждение `COMMENT ON EXTENSION postgis` — норма, не ошибка.)

### 2. Настроить backend

```powershell
cd backend
Copy-Item .env.example .env
```

В `backend\.env` вписать строку подключения с вашим паролем из шага 1:

```
DATABASE_URL=postgresql+psycopg://dubaicost:ПридумайтеПароль@localhost:5432/dubaicost
```

Установить зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 3. Запустить backend

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Проверка: открыть http://localhost:8000/docs — должна открыться страница API.
Это окно оставить открытым.

### 4. Запустить frontend (новое окно PowerShell)

```powershell
cd C:\путь\к\3D_dubaicost\frontend
npm install
npm run dev
```

### 5. Открыть карту

http://localhost:5173 — 3D-карта Дубая с переключателем слоёв.

## Путь Б — собрать данные с нуля (без дампа)

Нужен, если хотите обновить данные из источников. DLD-выгрузки должны лежать в
`backend\raw\` (см. `scripts\fetch_official_data.py` для загрузки внутри ОАЭ).

```powershell
# создать пустую БД с PostGIS (вручную через psql), затем:
cd backend
Copy-Item .env.example .env    # прописать DATABASE_URL
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe ..\scripts\bootstrap_local.py
```

Скрипт по порядку: миграции → здания из OSM → cooling-провайдер → коннекторы
DLD + агрегация → парковка → демо-слой → POI. Шаги идемпотентны.

## Проверка, что всё работает

- Карта открывается, здания видны в 3D.
- Слой **«Цена»** — здания в Marina / Downtown / Business Bay / JLT окрашены.
- Переключение слоёв (Аренда, Service charge, Cooling, Парковка, Год) — без
  ошибок, серые здания = «нет данных» (это норма, не ноль).
- Клик по зданию — карточка с расчётом стоимости.
- Тумблеры «Диаспоры по районам» и «Достопримечательности» — включаются.

## Обновление данных (опционально, позже)

- Разовый пересчёт: `.\.venv\Scripts\python.exe ..\scripts\bootstrap_local.py`
- Новый дамп для передачи:
  `pg_dump -U dubaicost -h 127.0.0.1 -d dubaicost -Fc -f delivery\dubaicost.dump`
- Ночное авто-обновление (cron 03:00) описано в `README.md`.

## Если понадобится публичная ссылка

Сейчас всё работает на вашем компьютере (localhost) — ссылки для сторонних нет.
Чтобы показывать карту по URL, разворачивается на VPS тем же способом
(`docker compose up`, см. `README.md`), бюджет ~$15–40/мес.
